# Excel Anonymizer

Приложение обезличивает Excel-файлы и умеет расшифровывать ранее обезличенные файлы по сохраненному `mapping.xlsx`.

## Что делает приложение

- удаляет гиперссылки из ячеек, не меняя отображаемый текст;
- заменяет ФИО на новые значения;
- скрывает логины в формате `name.surname`;
- создает отдельный файл с расшифровкой соответствий.
- умеет восстановить исходные значения по этому файлу соответствий.

## Что получается на выходе

Если выбран файл `report.xlsx`, рядом будет создана папка:

```text
report_anonymized/
```

Внутри нее будут два файла:

- `report_anonymized.xlsx` — обезличенный файл;
- `report_mapping.xlsx` — расшифровка замен ФИО и логинов.

После расшифровки обезличенного файла внутри папки с обезличенным файлом будет создана папка:

```text
report_anonymized/report_anonymized_decrypted/
```

Внутри нее появится файл:

- `report_anonymized_decrypted.xlsx` — восстановленный файл.

## Как запустить

Из папки проекта:

```bash
./venv/bin/python main.py
```

После запуска:

- откроется окно выбора действия: обезличить или расшифровать файл;
- откроется окно выбора Excel-файла на macOS;
- в режиме расшифровки `mapping.xlsx` будет найден автоматически рядом с файлом, а если его нет, откроется отдельный выбор этого файла;
- после завершения можно сразу открыть готовый файл или папку с результатом.

По умолчанию запуск без аргументов работает как оконный режим без `input()` и без работы в терминале.

Также можно сразу передать путь к файлу:

```bash
./venv/bin/python main.py data.xlsx
```

CLI-режим по-прежнему доступен для автоматизации и тестов.

## Как расшифровать файл

Если рядом с обезличенным файлом уже есть `*_mapping.xlsx`, достаточно запустить:

```bash
./venv/bin/python main.py --decrypt report_anonymized/report_anonymized.xlsx
```

Если файл соответствий лежит в другом месте, передай его явно:

```bash
./venv/bin/python main.py --decrypt report_anonymized/report_anonymized.xlsx --mapping-file /path/to/report_mapping.xlsx
```

## Как проверить результат

1. Открой файл `*_anonymized.xlsx`.
2. Убедись, что текст в ячейках сохранился.
3. Проверь, что гиперссылки больше не кликаются.
4. Проверь, что ФИО заменены одинаково во всех повторяющихся местах.
5. Проверь, что логины вида `name.surname` заменены.
6. Открой файл `*_mapping.xlsx` и проверь таблицу соответствий.
7. При необходимости запусти режим `--decrypt` и проверь, что исходные ФИО и логины восстановились.

## Требования

- macOS для выбора файла через системное окно;
- Python из `venv`;
- библиотека `openpyxl`.

## Структура

- `main.py` — точка входа, выбирает GUI- или CLI-режим;
- `anonymizer_core.py` — вся логика обработки Excel;
- `macos_ui.py` — оконный интерфейс через `osascript`.

## Пример

Если в файле есть:

- `Иван Петров`
- `ivan.petrov`
- ячейка с текстом и гиперссылкой

то после обработки:

- ФИО будет заменено на новое;
- логин будет заменен на технический псевдологин;
- текст ячейки останется прежним, но гиперссылка будет удалена;
- в отдельном файле появится расшифровка замен.

Если затем запустить `--decrypt` с этим `mapping`-файлом, исходные ФИО и логин будут восстановлены в новом Excel-файле.
